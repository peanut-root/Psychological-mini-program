# Feature Specification: Floating AI Agent Ball

**Feature Branch**: `002-floating-agent`  
**Created**: 2026-04-07  
**Status**: Draft  
**Input**: User description: "i need to write an agent act as a floating ball on the edge of the screen, inserting the api (please take the place for apikey empty)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 悬浮球展示与拖动 (Priority: P1)

用户在小程序的任意主要页面中，屏幕边缘始终显示一个可拖动的悬浮球。悬浮球可吸附在屏幕左右边缘，不遮挡核心内容。

**Why this priority**: 悬浮球是用户与AI智能体交互的唯一入口，是最基础的视觉与交互层。

**Independent Test**: 仅检查悬浮球是否显示、是否可拖动、是否吸附边缘即可验证本故事，独立于AI对话功能。

**Acceptance Scenarios**:

1. **Given** 用户进入任一挂载了悬浮球的页面， **When** 页面渲染完成， **Then** 屏幕边缘出现悬浮球且不遮挡页面主要内容。
2. **Given** 用户按住悬浮球拖动， **When** 松开手指， **Then** 悬浮球自动吸附到最近的屏幕左右边缘。
3. **Given** 悬浮球吸附在边缘， **When** 用户点击悬浮球， **Then** 触发AI对话面板展开（见 User Story 2）。

---

### User Story 2 - AI 对话面板展开与交互 (Priority: P2)

用户点击悬浮球后，展开一个对话面板，用户可输入文字消息并获得AI回复。

**Why this priority**: 悬浮球的核心价值在于提供随时可访问的AI对话能力。

**Independent Test**: 仅通过模拟API回复即可验证对话面板的展示、消息发送与接收，不依赖真实API连通性。

**Acceptance Scenarios**:

1. **Given** 悬浮球处于边缘收起状态， **When** 用户点击悬浮球（移动距离 < 10rpx）， **Then** 对话面板从屏幕右侧滑出（宽度 75%，时长 ≤500ms），展示历史对话记录。
2. **Given** 对话面板已展开， **When** 用户输入文字并发送， **Then** 消息显示在面板中，且AI回复在 API Key已配置时 ≤5 秒内出现、未配置时 1 秒内出现模拟回复。
3. **Given** 对话面板已展开， **When** 用户点击关闭按钮或悬浮球， **Then** 面板收起（时长 ≤300ms），悬浮球回到边缘。

---

### User Story 3 - API 接入与对话能力 (Priority: P3)

悬浮球的AI对话通过外部大模型API实现，API Key预留为占位符，后续可填入真实密钥。

**Why this priority**: 真实AI能力依赖API接入，但面板与交互可先用模拟数据验证。

**Independent Test**: 使用模拟回复数据验证对话流程，随后切换为真实API Key验证端到端连通性。

**Acceptance Scenarios**:

1. **Given** API Key未配置（占位符）， **When** 用户发送消息， **Then** 系统返回模拟回复或提示配置API Key。
2. **Given** API Key已正确配置， **When** 用户发送消息， **Then** 请求发送至API并在合理时间内返回AI回复。

---

### Edge Cases

- **页面覆盖范围**：悬浮球不挂载在引导页（splash/splash2）等全屏沉浸式页面，需明确挂载范围。
- **网络异常**：API请求超时或失败时，对话面板应展示友好的错误提示而非白屏。
- **键盘遮挡**：移动端输入时虚拟键盘弹出，对话面板应自动调整高度避免被键盘遮挡。
- **多页面切换**：用户在不同页面间切换时，悬浮球位置保持一致，对话面板自动收起。
- **超长对话内容**：对话历史过长时需支持滚动加载，避免内存占用过高。
- **AI 超长回复**：AI 返回的文本超过面板可视区域时，消息气泡必须支持内部滚动，不得溢出或截断。
- **连续快速发送**：用户在前一条回复未到达前连续发送多条消息，系统必须按序排队请求，不得丢弃或合并消息。
- **API 响应中断**：请求发送后网络连接断开（mid-response），系统必须展示"响应中断，请重试"提示，保留用户已发送的消息。
- **用户打开聊天面板时进入非挂载页面**：面板自动收起，悬浮球保持当前位置。
- **本地存储损坏**：若读取保存的位置或对话历史失败，系统必须使用默认位置（右边缘 400rpx）和空对话历史，不得崩溃。
- **消息发送失败文本恢复**：发送失败后，用户输入框中的文本必须保留，不可清空。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在以下目标页面的屏幕边缘渲染一个悬浮球组件：首页（home）、文献页（literature）、搜索页（search）、个人中心（my-page）、OCD页面（ocd）。悬浮球不随页面滚动而消失，不遮挡页面主要内容（悬浮球中心距离屏幕上下边缘至少 120rpx，距离左右边缘不超过 20rpx 当吸附时）。
- **FR-002**: 用户必须能够按住并拖动悬浮球在屏幕可视区域内自由移动，拖动时悬浮球中心坐标（x, y）的约束范围为：x ∈ [球半径, 屏幕宽度-球半径]，y ∈ [状态栏高度+球半径, 屏幕高度-球半径]。松开后，若悬浮球中心距屏幕左边缘或右边缘的距离 < 屏幕宽度的 40%，则自动吸附到该边缘（球边缘距屏幕边缘 12rpx）；否则回到原吸附侧。
- **FR-003**: 用户点击悬浮球（移动距离 < 10rpx 视为点击，而非拖动）必须展开/收起AI对话面板。面板从屏幕右侧滑入滑出，宽度为屏幕宽度的 75%，动画时长 ≤500ms，缓动函数为 ease-out。面板展开时悬浮球保持在面板外侧边缘可见，作为关闭按钮的替代操作。
- **FR-004**: 对话面板必须展示消息列表（用户消息与AI回复），支持滚动浏览历史记录。
- **FR-005**: 用户必须能够在对话面板中输入文字消息并点击发送按钮。
- **FR-006**: 系统必须通过外部AI服务获取回复，服务密钥以占位符形式存放在配置中，便于后续填入真实密钥。
- **FR-007**: 当API Key未配置（仍为空字符串）时，系统必须在 1 秒后返回模拟回复"AI 服务配置中，请稍后填写 API Key"，不可直接崩溃或无响应。
- **FR-008**: API请求超时或失败时，对话面板必须展示错误提示并允许用户重试。
- **FR-009**: 虚拟键盘弹出时，对话面板的输入区域必须保持在键盘上方可见。
- **FR-010**: 悬浮球位置（x, y, 吸附侧）在页面切换时必须保持一致，通过本地存储持久化。对话面板的展开/收起状态在页面切换时自动重置为收起（不跨页面保持打开）。对话历史记录在同一设备上的不同页面间共享。
- **FR-011**: 悬浮球图标尺寸固定为 96rpx × 96rpx，圆角 50%（圆形），z-index 高于页面内容但低于对话面板。首次加载时若本地存储无位置数据，默认吸附在屏幕右边缘、距顶部 400rpx 处。
- **FR-012**: 对话面板必须包含以下结构区域：顶部标题栏（显示"AI 心理助手" + 关闭按钮 ×）、中间消息滚动区（scroll-view）、底部输入区（textarea + 发送按钮）。面板背景色为纯白（#FFFFFF），带左侧 4rpx 阴影。
- **FR-013**: 系统必须区分"点击"（touchstart 到 touchend 移动距离 < 10rpx）和"拖动"（移动距离 ≥ 10rpx）行为。点击触发面板展开/收起，拖动更新悬浮球位置。
- **FR-014**: 悬浮球在拖动过程中的帧率必须保持 ≥30fps，不得出现卡顿或白屏闪烁。
- **FR-015**: 系统必须确保 API Key 仅保存在内存变量中，不得通过日志输出、网络请求（除 API 调用本身外）或本地存储泄露。
- **FR-016**: 对话面板在展开时必须阻止页面底层内容的滚动和触摸事件，收起时恢复。
- **FR-017**: 悬浮球图标必须设置 `alt` 等效 WXML `aria-label` 属性为"AI 心理助手"，关闭按钮的 `aria-label` 为"关闭对话面板"。
- **FR-018**: 对话面板中的文本必须使用 ≥28rpx 字号，行高 ≥1.6，确保在系统大字模式下仍可正常显示。
- **FR-019**: AI 回复内容若涉及医疗诊断建议，必须在消息底部追加系统现有的心理健康免责声明链接或提示。
- **FR-020**: AI 服务请求必须在系统提示词（system prompt）中包含以下约束："你是一个支持性心理健康助手，不提供医疗诊断。请倾听、理解、共情，必要时建议用户寻求专业帮助。"
- **FR-021**: 对话历史最多保存 50 条消息（用户+AI回复各计一条），超出时自动删除最早的消息（FIFO）。对话历史在小程序缓存被清除时一并清除。悬浮球位置缓存在缓存清除时一并清除。
- **FR-022**: 用户必须能够在对话面板中点击"清除历史"按钮，清空所有本地对话记录。
- **FR-023**: 发送失败时，用户消息必须保留在输入框中供用户修改重试，而非清空后丢失。

### Key Entities

- **FloatingBall**: 悬浮球组件，使用彩色条纹球体图标（`images/floating-ball-icon.png`）作为视觉呈现，固定尺寸 96rpx × 96rpx，圆角 50%。包含位置坐标（x, y）、吸附状态（left/right）、拖动中状态。默认位置为屏幕右边缘距顶 400rpx。位置信息持久化至本地存储，缓存清除时一并清除。
- **ChatPanel**: 对话面板，宽度为屏幕宽度 75%，从右侧滑入滑出（≤500ms ease-out）。包含顶部标题栏（"AI 心理助手" + 关闭按钮）、中间 scroll-view 消息区、底部输入区（textarea + 发送按钮）。背景纯白（#FFFFFF），左侧 4rpx 阴影。展开时覆盖页面右侧内容，不遮挡悬浮球（悬浮球在面板外侧可见）。
- **ChatMessage**: 单条对话消息，包含角色（user/assistant/system）、内容文本、时间戳（Unix ms）、发送状态（pending/sent/error）。用户消息靠右对齐，AI回复靠左对齐。
- **APIConfig**: API连接配置，包含 API Key（空字符串占位符，仅存内存）、API端点URL、超时时间（8000ms）、模型名称、系统提示词。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% 目标页面挂载悬浮球且首屏渲染无闪烁（抽检全部目标页面）。
- **SC-002**: 拖动悬浮球后松开，100% 情况下悬浮球在 300ms 内完成边缘吸附动画。
- **SC-003**: 用户从点击悬浮球到对话面板完全展开的过渡时间不超过 500ms。
- **SC-004**: API Key已配置时，90% 的对话请求在 5 秒内收到AI回复。
- **SC-005**: 网络异常或API Key未配置时，100% 情况下用户看到明确的提示信息而非空白/错误页面。
- **SC-006**: 虚拟键盘弹出时，输入区域 100% 可见不被遮挡。

## Assumptions & Dependencies

### Assumptions

- **A-001**: 悬浮球仅在小程序的主要功能页面展示（首页、文献、搜索、个人中心、OCD），不展示在引导页（splash/splash2）等全屏沉浸式页面，也不展示在文献详情页、医院页、量表页等子页面。
- **A-002**: AI 服务为第三方外部服务，其可用性不受本小程序控制；需假设服务可用率 ≥99%。
- **A-003**: 用户为单设备单会话使用场景，对话历史仅绑定当前设备，不跨设备同步。
- **A-004**: AI回复内容不涉及医疗诊断建议，仍需遵守小程序现有的心理健康内容免责声明。
- **A-005**: 现有的 `pages/ai/ai` 独立页面保持现状，不与悬浮球功能冲突，两者可共存。用户可从 home 页原有入口进入独立 AI 页面，也可通过悬浮球使用浮层对话。
- **A-006**: 微信开发者工具的调试环境可模拟网络断开和延迟场景，用于手动测试。

### Dependencies

- **D-001**: 外部AI大模型服务提供商的API接口需兼容 OpenAI 格式的 Chat Completion API（POST /v1/chat/completions，请求体包含 `model`, `messages`, `temperature`）。候选服务包括但不限于 OpenAI、DeepSeek、通义千问等。
- **D-002**: API Key的获取与配置流程由项目负责人另行提供，本功能仅预留占位符。
- **D-003**: 小程序运行环境需支持 `wx.request` 或等效的网络请求能力。所选 AI 服务域名必须在微信小程序管理后台的"服务器域名"白名单中配置。
